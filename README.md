# NdcHostEmulator

TCP-сервер для эмуляции NDC-хоста. Принимает подключения, отправляет NDC-команды клиенту и логирует весь трафик. Доступен в двух вариантах — консольном и веб.

## Консольная версия

Интерактивный UI на базе [Spectre.Console](https://spectreconsole.net/).

```bash
dotnet run --project NdcHostEmulator.Console
```

При запуске запрашивает порт (по умолчанию `4070`, сохраняется в `state.json`). После подключения клиента доступно меню:

- **Отправить файл** — выбор из директории `Files/`, отправка с разбивкой по `[FIELD]`
- **Отправить текст** — ввод вручную, поддерживает `[FIELD]`; управляющие символы вводятся через Unicode-символы (например `␜` → байт `0x1C`)
- Пауза/возобновление логирования входящих данных
- Просмотр и очистка лога (`tcp_sender.log`)
- Отключение клиента без остановки сервера

## Веб-версия

Blazor Server + MudBlazor, управление через браузер.

```bash
dotnet run --project NdcHostEmulator.Web
```

- Порт и директория файлов настраиваются прямо в интерфейсе и сохраняются в `state.json`
- **Отправить файл** — выбор из директории, разбивка по `[FIELD]`
- **Отправить текст** — с поддержкой `[FIELD]`
- Лог в реальном времени прямо в браузере (последние 500 событий)
- Логирование в файл через Serilog с ротацией по дням

## Возможности

- Принимает одно TCP-подключение за раз
- Отправка файлов из директории `Files/` и произвольного текста
- Поддержка разделителя `[FIELD]` — каждая часть отправляется отдельным NDC-сообщением
- Все сообщения отправляются с 2-байтовым заголовком длины: `[HIGH][LOW][...данные...]`
- Управляющие символы в тексте вводятся через Unicode-символы (␜ → `0x1C` и т.д.)
- Логирование всех событий (входящие, исходящие, подключение/отключение)

## Конфигурация

`appsettings.json`:

```json
{
  "FilesDirectory": "./Files"
}
```
