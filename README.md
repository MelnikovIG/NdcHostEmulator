# NdcHostEmulator

Консольный TCP-сервер для эмуляции NDC-хоста. Позволяет принимать входящие TCP-подключения, отправлять файлы и произвольные данные клиенту, а также вести лог всей сетевой активности.

---

## Возможности

- 🔌 **TCP-сервер** — принимает одно подключение за раз на указанном порту
- 📁 **Отправка файлов** — отправляет файлы из настроенной директории по протоколу с длиной-заголовком (2 байта)
- 📤 **Произвольные данные** — отправка текста или HEX-последовательностей вручную
- 📥 **Мониторинг входящих данных** — асинхронное чтение и логирование всех данных от клиента
- 📋 **Логирование** — запись всех событий в файл `tcp_sender.log` с возможностью паузы/возобновления
- 💾 **Сохранение состояния** — последний использованный порт сохраняется в `state.json`
- 🎨 **Красивый интерфейс** — консольный UI на базе [Spectre.Console](https://spectreconsole.net/)

## Установка и запуск

```bash
git clone <repo-url>
cd NdcHostEmulator
dotnet run
```

При первом запуске будет предложено ввести порт (по умолчанию `4070`).

## Конфигурация

Настройки хранятся в файле `appsettings.json`:

```json
{
  "FilesDirectory": "./Files"
}
```

| Параметр         | Описание                                  | По умолчанию |
|------------------|-------------------------------------------|--------------|
| `FilesDirectory` | Путь к директории с файлами для отправки  | `./Files`    |

---

## Формат файлов

Файлы в директории `Files` могут содержать несколько команд, разделённых токеном `[FIELD]`.  
Каждая команда отправляется отдельно с 2-байтовым заголовком длины:

```
[HIGH_BYTE][LOW_BYTE][...данные команды...]
```

**Пример содержимого файла:**
```
COMMAND_ONE[FIELD]COMMAND_TWO[FIELD]COMMAND_THREE
```

## Меню управления

После подключения клиента доступно интерактивное меню:

| Пункт                        | Описание                                        |
|------------------------------|-------------------------------------------------|
| 📁 Выбрать и отправить файл   | Выбрать файл из директории и отправить клиенту  |
| 📤 Отправить произвольный текст | Ввести текст и отправить его как есть          |
| 📤 Отправить HEX данные       | Ввести данные в HEX-формате (`48 65 6C 6C 6F`)  |
| 🔄 Обновить список файлов     | Обновить отображение файлов в директории        |
| 📊 Статус соединения          | Показать информацию о текущем подключении       |
| ⏸️ Пауза логгирования         | Остановить чтение и логирование входящих данных |
| ▶️ Возобновить логгирование   | Возобновить чтение входящих данных              |
| 📝 Показать логи              | Вывести последние 20 записей из лога            |
| 🧹 Очистить логи              | Удалить файл `tcp_sender.log`                   |
| ❌ Отключить клиента           | Закрыть соединение с текущим клиентом           |
| 🚪 Выйти из программы         | Завершить работу сервера                        |

## Структура проекта

```
NdcHostEmulator/
├── Program.cs           # Основной код приложения
├── appsettings.json     # Конфигурация
├── state.json           # Сохранённое состояние (порт)
├── tcp_sender.log       # Файл логов (создаётся автоматически)
└── Files/               # Директория с файлами для отправки
```

---

## Логирование

Все события записываются в `tcp_sender.log` в формате:

```
[HH:mm:ss.fff] [TYPE] Сообщение
```

Типы событий: `CONNECT`, `DISCONNECT`, `INCOMING`, `OUTGOING`, `DATA`, `HEX`, `ERROR`, `SYSTEM`.

---

## Ограничения

- Сервер обрабатывает **только одно подключение одновременно**. После отключения клиента сервер ожидает следующего.
- Входящие данные логируются, но не обрабатываются автоматически.
